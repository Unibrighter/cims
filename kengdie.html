第二步:安装egit

在myeclipse中选择MyEclipse-->MyEclipse Configration Center-->software-->上搜索egit选择collaboraton中的eclipse egit安装。

也可以去egit的官网找，不过貌似不提供下载连接。

配置egit。这一步与git配置略重复。

打开Preferences > Team > Git > Configuration，点击 New Entry ，新建一个键值对

key(user.name)value("your name")和key(user.email)("your email")

这个操作和git bash中 git config –global user.name “your name” 和 git config –global user.email you@example.com的配置是一样的

打开环境变量，新建一个HOME，变量值输入"%USERPROFILE%"或者其他一个路径

注册github

这个不多谈，访问不了翻墙。https://github.com

之后最好先自己 create a repo体验一下。

eclipse上讲一个项目push到自己建的repo中

选择一个工程，点击右键，Team->Share Project，选择git

选择Use or create repository in parent folder…,进入选择Create Repository对这个工程初始化一个git版本库，如果这个工程已经在于一个版本库的工作树上的话，这个版本库会自动选择.也可以不选择这个，找个地方存着。

对工程点击右键，选择team，会出来git的命令，选择commit，然后选择你要提交的文件，加上提交留言，点击commit，完成提交。

这样将项目commit到了本地

然后选择team--remote--push,在Destination Git Repository中url填你创建的那个repo的https地址。这里不用网上的ssh方法，那个麻烦一些。然后next

点那个Add All brench spec,next

这时候继续几步next貌似，不出现错误的话就成功了。

更有可能环境还没配好。

出现问题的话，自己查一下吧，我也忘了有哪些问题了，到时候去google。或者问我，我应该也遇到过错误

如果出现Non-fast-forward错误。大概意思是git仓库中已经有一部分代码，所以它不允许你直接把你的代码覆盖上去。

方法一：强行push。（不推荐）

方法二：先pull下来，是本地版本和remote端一样，再push。
